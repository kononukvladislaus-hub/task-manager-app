# 📋 Task Manager (Менеджер задач)

Полнофункциональное веб-приложение (SPA) для создания, редактирования, фильтрации и удаления задач.

**Разработчик:** Кононюк Владислав Алексееевич 25ТИДИ
**Стек:** Python 3.11, Flask, SQLite, Vanilla JS (ES6), Docker, Pytest, Semgrep, Trivy.

---

## 🏗️ 1. Архитектура и Структура системы

Проект построен по классической клиент-серверной схеме с разделением данных и логики обработки.

* **Backend:** RESTful API на фреймворке Flask (`app.py`). Обрабатывает входящие HTTP-запросы, валидирует данные (длина заголовка, пустота строк) и взаимодействует с СУБД.
* **Database:** Реляционная БД SQLite (`tasks.db`). Состоит из таблицы `tasks` со следующей схемой:
    * `id` (INTEGER, PRIMARY KEY, AUTOINCREMENT)
    * `title` (TEXT, NOT NULL)
    * `description` (TEXT)
    * `completed` (INTEGER, DEFAULT 0)
    * `created_at` (TIMESTAMP)
* **Frontend:** Клиентская часть написана на чистом JavaScript (`script.js`) без внешних тяжелых фреймворков. Взаимодействие с сервером происходит асинхронно через API (`Fetch API`). Для безопасности вывода данных от XSS-атак используется экранирование (`escapeHtml`).
* **Инфраструктура:** Приложение полностью контейнеризировано с помощью `Dockerfile` на базе легковесного образа `python:3.11-slim`. Конфигурация `docker-compose.yml` настраивает постоянное хранение данных (Persistent Volume) для предотвращения потери файлов базы данных.

🧪 3. Тестирование и Качество кода (QA)

В проекте реализовано автоматизированное функциональное тестирование API с помощью фреймворка pytest (tests/test_app.py).

    Изоляция среды: Во время тестов база данных разворачивается непосредственно в оперативной памяти (:memory:), исключая влияние тестов на реальные данные пользователей.

    Запуск тестов: pytest -v

🛡️ 4. Безопасность и Девопс пайплайн (SAST / SCA)

Интегрирован современный подход DevSecOps. В репозитории настроен пайплайн автоматической проверки кода при каждом Pull Request (.github/workflows/ci.yml):

    SAST (Анализ кода): Инструмент Semgrep проверяет исходный код на наличие антипаттернов. Защита от SQL-инъекций гарантирована на уровне архитектуры за счет использования параметризованных SQL-запросов (плейсхолдеры ?).

    SCA (Анализ зависимостей): Сканер Trivy проверяет сторонние библиотеки из requirements.txt на известные уязвимости (CVE) и генерирует SBOM-отчет формата CycloneDX (sbom.json).
